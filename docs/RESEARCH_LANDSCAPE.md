# Research Landscape

## Current Category Shape

The broad "deep research" and agent-framework layers already exist.

That means Noeris should not try to win by being:

- another generic browsing-and-synthesis app
- another general agent runtime
- another vague multi-agent framework

## Real Opening

The stronger opening is:

- structured research memory
- bounded experiment generation
- verification-rich execution loops
- evidence-backed memos with replayable artifacts

In other words: trustworthy discovery, not just research-flavored automation.

## Implication For Architecture

This is why the system is shaped as:

- source collection
- research memory
- hypothesis planning
- experiment planning
- execution
- verification
- memo writing

The memo is downstream of the loop. It is not the loop.

## Implication For Scope

The company vision can be broad, but the first product should stay narrow:

- ML/LLM research first
- post-training and evals first
- public papers, repos, and benchmarks first

This keeps the feedback loop fast and the evidence legible.

## What To Defer

Until the core loop is working, avoid:

- general multi-domain science claims
- pre-training-heavy roadmaps
- autonomous publication ambitions
- heavyweight orchestration or infra theater

## Practical Read

The early moat is not "having agents."

The early moat is:

- better research memory
- tighter verification
- clearer replay
- better judgment about what to test next
